# Acceptance Criteria

## Allgemein

- Die App startet lokal ohne manuelle Sondertricks.
- Alle Hauptrouten sind erreichbar.
- Der Code ist modular und verständlich.
- Kein chaotischer Mischstil im UI.

## Upload und Library

- Ein Bild kann hochgeladen werden.
- Es erscheint in einem Projekt oder in der Library.
- Es gibt ein Thumbnail.
- Der Hover zeigt Download und Edit.
- Download liefert die Datei tatsächlich aus.

## Editor environment_edit

- Ein Bild kann im Editor geöffnet werden.
- Stil und Licht sind wählbar.
- Eine Generierung kann ausgelöst werden.
- Das Ergebnis wird gespeichert.
- Das Ergebnis erscheint als neue Variante.

## material_edit

- Materialmodus ist im UI auswählbar.
- Generierung nutzt den richtigen Prompt-Modus.
- Varianten werden korrekt gespeichert.

## room_insert

- Ein Raumfoto kann hochgeladen werden.
- Ein Möbelbild kann gewählt werden.
- Klickposition wird erfasst.
- Optional kann eine Zielgröße gezogen werden.
- Eine Generierung kann ausgelöst werden.
- Ergebnis wird gespeichert.

## Kosten

- Es gibt einen eigenen Kosten-Screen.
- Summary Cards zeigen Werte.
- Tabellenansicht funktioniert.

## Design

- Buttons/Inputs haben 2px Inset Shadow.
- Keine schweren Drop Shadows.
- Bauhaus-Farben werden konsistent eingesetzt.
