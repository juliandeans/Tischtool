# UX- und UI-Konzept

## Grundgefühl

Die App soll ruhig, direkt und werkzeughaft wirken.

Nicht wie ein kreativer Chatbot.
Nicht wie ein Pinterest-Spielzeug.
Nicht wie ein Enterprise-Dashboard.

## Navigation

Linke Sidebar:

- Projekte
- Library
- Editor
- Raumfoto einsetzen
- Presets
- Kosten
- Einstellungen

## Hauptbildschirm

Das Bild ist immer der Mittelpunkt.

Einstellungen sitzen rechts in einer kompakten Sidebar.

## Editor-Modi

### Umgebung ändern

Fokus:
Hintergrund, Wand, Licht, Atmosphäre, kleine Zusätze.

### Material ändern

Fokus:
Holzart, Oberfläche, Materialcharakter.

### Raumfoto einsetzen

Fokus:
Hintergrundfoto + Möbelbild + Klickplatzierung.

## Library

Die Library ist Arbeitsverlauf und nicht nur Galerie.

Jede Bildkarte zeigt:

- Thumbnail
- Bildname
- Projekt
- Zeit
- Status optional

Hover oben rechts:

- Download
- Edit

## Editor-Eingaben

- Modus
- Stil-Preset
- Licht-Preset
- Variantenanzahl
- Zusatzhinweise
- Standardschutzregeln

## Standardschutzregeln

Standardmäßig aktiv:

- Objekt erhalten
- Perspektive erhalten
- Bildausschnitt möglichst erhalten
- keine zusätzlichen Möbel
- Änderungen primär an der Umgebung

## Raumfoto-Modus

Ablauf:

1. Raumfoto hochladen
2. Möbelbild hochladen
3. Klick in Raumfoto
4. optional Größe ziehen
5. Variante generieren

## UX-Prioritäten

1. schneller erster Erfolg
2. nachvollziehbarer Bildverlauf
3. wenige, gute Entscheidungen
4. kein Prompt-Wirrwarr
